  <what-built>
Phase 27 — Inventory Variant System (Colors & Sizes):
- Brand field moved directly below Name in add/edit form
- Type and Brand inputs have datalist autocomplete from existing inventory values
- "Has Colors" checkbox: enables per-color quantity rows (hides flat Color field)
- "Has Sizes" checkbox (only enabled when Has Colors is on): adds size sub-rows per color
- Variant items save has_colors, has_sizes, variants[], stock (sum) to Firestore
- Edit mode pre-fills color/size rows from stored variants
- Inventory list shows "N colors" or "N colors × M sizes" badge
- Billing: tapping a variant item opens a color/size picker sheet
- Excel export: variant items expand to one row per variant
  </what-built>
  <how-to-verify>
**Setup:** Use the app at your GitHub Pages URL (or local serve) and sign in as owner.

**Form changes:**
1. Go to Inventory → tap "+ Add Item"
2. Confirm Brand field appears directly below Name (second field)
3. Type a value in Type — confirm it shows suggestions from existing items
4. Type a value in Brand — confirm it shows suggestions from existing items

**Has Colors (no sizes):**
5. Check "Has Colors" → confirm flat Color field hides, color-variant rows appear
6. Add 2 color rows: "Blue" qty=5, "Red" qty=3 → Save → confirm toast success
7. Check Firestore (or observe list): item shows "2 colors" badge, stock=8
8. Tap the item to Edit → confirm form pre-fills with Blue and Red color rows

**Has Colors + Has Sizes:**
9. Add a new item → check Has Colors → then check Has Sizes
10. Add color row "Blue", add size sub-rows "M"=2, "L"=3
11. Add color row "Red", add size sub-row "M"=1
12. Save → item shows "2 colors × 2 sizes" badge in list
13. Edit that item → confirm all color and size rows pre-fill correctly

**Billing picker:**
14. Go to Billing → find the "Blue/Red" item → tap it → confirm color picker sheet appears with "Blue" and "Red" buttons
15. Tap "Blue" → confirm cart shows "ItemName (Blue)"
16. Find the color+size item → tap → confirm grouped picker by color, with size buttons under each
17. Tap a size → item added to cart with "ItemName (Blue / M)"

**Export:**
18. Go to Export → tap "Inventory" → open the downloaded xlsx
19. Confirm the variant item has multiple rows (one per variant) with Color and Size columns filled
20. Confirm flat items have a single row with empty Size column

**Regression check:**
21. Add a flat item (no Has Colors, no Has Sizes) → saves and displays as before
22. If there are existing Phase-13 hasSizes items (size+color+width+psi style), confirm they still display their "N sizes" badge and open the old edit modal correctly
  </how-to-verify>
