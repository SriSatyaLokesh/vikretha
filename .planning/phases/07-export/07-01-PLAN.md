Create modules/export.js from scratch with the following structure:

**Imports:**
```javascript
import { db } from '../lib/firebase-init.js';
import {
  collection, getDocs, query, where, orderBy, Timestamp
} from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js';
import { SHOP_ID, SHOP_NAME, LOCALE } from '../shop.config.js';
```

**Module-level state:**
```javascript
let _XLSX = null;  // cached SheetJS reference after first load
```

**`_loadSheetJS()` — lazy CDN loader:**
- Check `if (_XLSX) return _XLSX;`
- If `window.XLSX` is already defined (loaded elsewhere), use it directly
- Otherwise dynamically inject `<script src="https://cdn.sheetjs.com/xlsx-0.20.3/package/dist/xlsx.full.min.js">` into `document.head`
- Wrap in a Promise, resolve onload, reject onerror with `new Error('Failed to load SheetJS. Check network.')`
- After load, cache: `_XLSX = window.XLSX; return _XLSX;`

**`_showProgress(container, msg)` — progress overlay:**
- Find or create `#export-progress-overlay` div inside container
- Set class `export-progress-overlay` (styled in CSS — full-screen semi-transparent)
- Inner HTML: `<div class="export-progress-box"><div class="export-progress-spinner"></div><p>{msg}</p></div>`
- Set `display: flex`

**`_hideProgress(container)`:**
- Find `#export-progress-overlay` and set `display: none`

**`_safeStr(v)` — Excel formula injection prevention:**
- Convert value to string
- If starts with `=`, `+`, `-`, `@`, prefix with a space `' '`
- Return safe string

**`_exportSalesMonth(container, year, month)` — async:**
1. `const XLSX = await _loadSheetJS()`
2. `_showProgress(container, 'Fetching sales…')`
3. Build Firestore date range: `monthStart = new Date(year, month, 1)`, `monthEnd = new Date(year, month + 1, 0, 23, 59, 59, 999)`
4. Query: `query(collection(db,'shops',SHOP_ID,'sales'), where('timestamp','>=',Timestamp.fromDate(monthStart)), where('timestamp','<=',Timestamp.fromDate(monthEnd)), orderBy('timestamp'))`
5. `const snap = await getDocs(q)`
6. If `snap.size > 1000`: `_showProgress(container, 'Building Excel file…')`
7. Build rows array: header row + one row per sale
   - Header: `['Sale ID','Date','Items','Qty','Subtotal','Discount','Total','Phone']`
   - Per sale: `[sale_id, date_string, items_text, total_qty, subtotal, discount, total, phone]`
   - `items_text`: join items as `"Name×qty"` separated by `"; "` — use `_safeStr` on each name
   - `total_qty`: sum of all `item.quantity`
   - `date_string`: `s.timestamp?.toDate().toLocaleDateString(LOCALE)` or `''`
   - `phone`: `s.customer_phone || ''`
8. `const ws = XLSX.utils.aoa_to_sheet(rows)`
9. `const wb = XLSX.utils.book_new(); XLSX.utils.book_append_sheet(wb, ws, 'Sales')`
10. Filename: `{safeName}_sales_{year}-{paddedMonth}.xlsx` where `safeName = SHOP_NAME.replace(/[^a-zA-Z0-9_]/g,'_')`
11. `XLSX.writeFile(wb, filename)`
12. Wrap in try/finally: `_hideProgress(container)` always runs
13. On catch: alert with `'Export failed: ' + err.message`

**`_exportSalesAll(container)` — async:**
- Same as `_exportSalesMonth` but without date range filter (just `orderBy('timestamp')`)
- No year/month params; filename: `{safeName}_sales_all.xlsx`
- Otherwise identical logic

**`_exportInventory(container)` — async:**
1. `const XLSX = await _loadSheetJS()`
2. `_showProgress(container, 'Fetching inventory…')`
3. `const snap = await getDocs(collection(db, 'shops', SHOP_ID, 'inventory'))`
4. Header: `['ID','Name','Unit','Price','Stock','Threshold','Status']`
5. Per item: `[docId, name, unit, price, stock, threshold, status]`
   - `status`: `stock < (threshold ?? 5) ? 'Low Stock' : 'OK'`
   - Apply `_safeStr` to string fields
6. Sort rows (after header) alphabetically by Name before building sheet
7. Build sheet, workbook, filename: `{safeName}_inventory_{today_date}.xlsx`
   - `today_date`: `new Date().toISOString().slice(0,10)` → `YYYY-MM-DD`
8. try/finally with `_hideProgress`

**`attachExportMenu(container)` — exported entry point:**
1. Find `#dash-export-anchor` in container; if not found, return early silently
2. Build a button + dropdown HTML:
```html
<div class="export-menu-wrap">
  <button id="export-btn" class="export-btn" aria-haspopup="true" aria-expanded="false">
    <svg>...download icon...</svg>
    Export
  </button>
  <div id="export-dropdown" class="export-dropdown" role="menu" hidden>
    <button class="export-dropdown-item" data-export="sales-month">Sales (This Month)</button>
    <button class="export-dropdown-item" data-export="sales-all">Sales (All Time)</button>
    <button class="export-dropdown-item" data-export="inventory">Inventory</button>
  </div>
</div>
```
3. Inject into `#dash-export-anchor`
4. Toggle button: `#export-btn` click → toggle `hidden` on dropdown, toggle `aria-expanded`
5. Close on outside click: `document.addEventListener('click', handler)` — if click is outside `.export-menu-wrap`, hide dropdown and set `aria-expanded="false"`; store handler reference, remove on next call to `attachExportMenu` (use module-level `let _outsideClickHandler = null`)
6. Dropdown item click via delegated listener on `#export-dropdown`:
   - Hide dropdown immediately
   - Get `year, month` for "sales-month" from `container.querySelector('#dash-month-picker')?.value` — split by `-` → `[yearStr, monthStr]` → `parseInt` both. Fall back to current month if picker not found.
   - Call `_exportSalesMonth(container, year, month)`, `_exportSalesAll(container)`, or `_exportInventory(container)` based on `data-export`
   - Disable button during export, re-enable in finally

Use a download SVG icon (24×24 viewBox, stroke-based): `<path d="M21 15v4a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2v-4"/><polyline points="7 10 12 15 17 10"/><line x1="12" y1="15" x2="12" y2="3"/>`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Firestore → Excel cell | Firestore-stored strings (item names, sale IDs, phone numbers) flow into XLSX cells |
| CDN → browser | SheetJS script fetched from cdn.sheetjs.com at runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Excel cells | mitigate | `_safeStr()` prefixes any string starting with `=`,`+`,`-`,`@` with a space to prevent spreadsheet formula injection |
| T-07-02 | Tampering | SheetJS CDN | accept | CDN per project architecture decision (PROJECT.md). SheetJS is reputable; `cdn.sheetjs.com` is the official CDN. SRI not feasible with dynamic script injection. Risk accepted. |
| T-07-03 | Information Disclosure | .xlsx file | accept | File saved to device by user intent — this is the feature purpose. No PII beyond what user already entered. |
| T-07-04 | Denial of Service | _exportSalesAll | accept | Single-user app; Firestore Spark free-tier read limit (50K/day) is the natural cap. Progress overlay informs user for large datasets. |
</threat_model>
