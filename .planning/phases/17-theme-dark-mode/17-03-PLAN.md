<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
- `shop.config.js` — `THEME_PALETTES` constant: 6 palettes (Orange, Emerald, Sky, Violet, Rose, Slate)
- `styles/main.css` — 5 `[data-theme="..."]` palette blocks; `[data-dark="true"]` dark mode block; `.theme-swatch-grid`, `.toggle-btn` CSS
- `lib/firebase-init.js` — `applyTheme(themeId, dark)` export; `loadThemeFromFirestore()` export; IIFE applies localStorage-cached theme on every page load
- `app.js` — calls `loadThemeFromFirestore()` non-blocking after auth succeeds
- `modules/settings.js` — "Appearance" card: 6 palette swatches with active ring indicator; dark mode toggle with iOS-style animation; writes to Firestore `config/main` on every change
  </what-built>
  <how-to-verify>
Open the app in a browser and verify each item:

**Settings — Appearance card:**
1. Navigate to Settings → confirm "Appearance" section appears at the top of the settings list
2. Confirm 6 coloured circles are visible (orange, green, blue, purple, pink/rose, grey/slate)
3. The current active palette swatch has a visible ring/border indicator around it

**Palette switching:**
4. Tap the Emerald (green) swatch → all buttons, active nav items, and accent colours turn green instantly — no page reload
5. Tap the Violet (purple) swatch → app turns purple instantly
6. Tap the Sky (blue) swatch → app turns blue instantly
7. Tap the Rose swatch → app turns pink/red instantly
8. Tap the Slate swatch → app turns grey/dark-blue instantly
9. Tap Orange → returns to default orange
10. The active swatch always shows its ring indicator after selection

**Dark mode:**
11. Tap the dark mode toggle → entire app background goes dark, text turns light, cards turn dark-grey
12. Navigate to Billing, Inventory, Reports, Dashboard — all screens display correctly in dark mode
13. Tap dark mode toggle again → light mode restores correctly across all screens

**Combined palette + dark mode:**
14. Set palette to Violet + enable dark mode → confirm purple primary + dark backgrounds work together
15. Tap a different palette while in dark mode → palette changes without losing dark mode

**Persistence:**
16. Select Emerald palette + enable dark mode → hard-reload the page (Ctrl+R / Cmd+R)
17. Confirm the app loads with Emerald + dark mode already active (no flash of wrong theme)

**Firestore sync:**
18. After selecting a palette, wait 2 seconds, then open Firebase Console → Firestore → shops/{your_shop_id}/config/main → confirm `theme` and `darkMode` fields are written correctly
19. Open the app in a second browser tab → confirm it loads with the same theme (synced from Firestore)

**Correctness across all screens:**
20. In each palette + dark mode, navigate through: Dashboard, New Sale (Billing), Inventory, Reports, Settings — confirm no screen looks broken or unreadable

**No regressions:**
21. Add a staff email → save works normally
22. Record a test sale → billing works normally
23. No JavaScript console errors throughout the verification
  </how-to-verify>
