<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client billing form → Firestore | payment_mode and payment_split come from user input via billing.js |
| Firestore rule | Validates payment_mode is a string and is present on every new sale |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-04 | Tampering | payment_mode value from client | mitigate | Firestore rule validates `payment_mode is string`; client is authenticated via isAuthorized(); value set only via the billing.js UI flow |
| T-25-05 | Tampering | payment_split amounts — no server-side sum validation | accept | Split amounts are informational/audit only; total already validated independently; wrong amounts don't affect financial total stored on the sale |
| T-25-06 | Denial of Service | Malformed payment_split (null/non-map) | accept | paySplit is null for non-split modes (stored as null in Firestore, harmless); no server reads depend on its structure |
</threat_model>
