**Change 3 — WhatsApp: route directly to customer phone when available**

In the `btn-whatsapp` click handler, add a guard at the very top of the handler function BEFORE the `navigator.canShare` check:

```javascript
document.getElementById('btn-whatsapp').addEventListener('click', async () => {
    const message = `Receipt from ${SHOP_NAME}\nBill #${saleId}\nTotal: ${CURRENCY}${sale.total.toFixed(2)}`;

    // If the sale has a customer phone, route directly to that customer's WhatsApp.
    // Skip Web Share API — it opens a generic system sheet with no phone routing.
    if (sale.customer_phone) {
      const phone = sale.customer_phone.replace(/\D/g, '');
      const text  = encodeURIComponent(message);
      window.open(`https://wa.me/${phone}?text=${text}`, '_blank', 'noopener,noreferrer');
      return;
    }

    // No customer phone: try Web Share API (lets shopkeeper choose recipient)
    if (navigator.canShare) {
      // ... existing navigator.share logic unchanged ...
    }

    // Final fallback: shop WhatsApp number or generic wa.me
    const phone = (WHATSAPP_NUMBER || '').replace(/\D/g, '');
    const text  = encodeURIComponent(message);
    const url   = phone
      ? `https://wa.me/${phone}?text=${text}`
      : `https://wa.me/?text=${text}`;
    window.open(url, '_blank', 'noopener,noreferrer');
});
```

The existing `wa.me` fallback block at the bottom of the handler used `sale.customer_phone || WHATSAPP_NUMBER` — update it to `WHATSAPP_NUMBER` only since the customer_phone case is now handled by the early return above.
