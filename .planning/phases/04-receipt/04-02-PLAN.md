  <how-to-verify>
    **Pre-requisite:** Ensure the app is running/deployed and a sale exists (run a test sale
    through the billing screen first if needed).

    1. Complete a test sale via `#/billing`
    2. On the confirmation screen, tap **"View Receipt"**
    3. Verify: receipt page loads without error, canvas receipt image appears within 1 second
    4. Verify receipt image contains:
       - Shop name at the top
       - Date and Bill # (e.g. 20260531-0001)
       - All line items with qty, unit price, and line total
       - Subtotal, any discount, and grand total
    5. Tap **"Download Receipt"** — verify PNG file downloads to device/browser downloads
    6. Tap **"Share via WhatsApp"**:
       - On Android/Chrome: native share sheet should appear with the receipt image
       - On desktop: wa.me link should open in new tab with pre-filled message
    7. Tap **"← Back to Dashboard"** — verify returns to dashboard
    8. Direct URL test: navigate to `#/receipt/NONEXISTENT` — verify "Receipt not found" message
    9. Confirm no horizontal scroll at mobile viewport (≤ 430px wide)

    **Phase 4 UAT checklist (from ROADMAP.md):**
    - [ ] Receipt image generated in < 1 second after sale
    - [ ] Receipt displays all required fields (shop, date, Sale ID, items, totals)
    - [ ] PNG downloads correctly to device
    - [ ] WhatsApp share opens with correct message + image (on supported devices)
    - [ ] Fallback wa.me link works when Web Share API unavailable
    - [ ] Receipt renders correctly on mobile viewports
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe any issues found.</resume-signal>
