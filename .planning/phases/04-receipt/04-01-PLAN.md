### modules/receipt.js

Export a single `render(container, saleId)` function. All logic is inside.

**Imports:**
```js
import { db }          from '../lib/firebase-init.js';
import { doc, getDoc } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js';
import {
  SHOP_NAME, SHOP_ID, CURRENCY, LOCALE, LOGO_URL, WHATSAPP_NUMBER, THEME_COLOR
} from '../shop.config.js';
```

**render(container, saleId) flow:**

1. If `!saleId`, show error: `<div class="empty-state"><p>No receipt ID provided.</p></div>` and return.

2. Show loading spinner while fetching:
```html
<div style="padding:32px;text-align:center;"><div class="spinner" style="margin:0 auto;"></div></div>
```

3. `const saleRef = doc(db, 'shops', SHOP_ID, 'sales', saleId);`
   `const snap = await getDoc(saleRef);`
   If `!snap.exists()`: show `<div class="empty-state"><p>Receipt not found.</p><a href="#/dashboard">Back to Dashboard</a></div>` and return.

4. Extract sale data: `const sale = snap.data();`

5. Call `_drawReceipt(sale)` → returns `canvas` element (see below).

6. Build receipt page HTML in container:
```html
<div class="receipt-page">
  <!-- Receipt preview: show the canvas as image for mobile-friendly display -->
  <div class="receipt-preview">
    <!-- canvas element inserted here by JS, hidden after toDataURL -->
    <img id="receipt-img" alt="Receipt" style="max-width:100%;border-radius:8px;box-shadow:0 2px 12px rgba(0,0,0,0.12);" />
  </div>

  <div class="receipt-actions">
    <button id="btn-download" class="btn btn-primary btn-full">
      ↓ Download Receipt
    </button>
    <button id="btn-whatsapp" class="btn btn-full"
      style="background:#25D366;color:#fff;border:none;">
      Share via WhatsApp
    </button>
    <a href="#/dashboard" style="display:block;text-align:center;margin-top:4px;
       color:var(--text-secondary);font-size:0.875rem;">← Back to Dashboard</a>
  </div>
</div>
```

7. After rendering HTML, call `_drawReceipt(sale)`:
   - Draw to canvas
   - Convert: `const dataUrl = canvas.toDataURL('image/png')`
   - Set `document.getElementById('receipt-img').src = dataUrl`

8. Wire download button:
```js
document.getElementById('btn-download').addEventListener('click', () => {
  canvas.toBlob(blob => {
    const url = URL.createObjectURL(blob);
    const a   = document.createElement('a');
    a.href     = url;
    a.download = `receipt-${saleId}.png`;
    a.click();
    URL.revokeObjectURL(url);
  }, 'image/png');
});
```

9. Wire WhatsApp share button (Web Share API with wa.me fallback):
```js
document.getElementById('btn-whatsapp').addEventListener('click', async () => {
  const message = `Receipt from ${SHOP_NAME}\nBill #${saleId}\nTotal: ${CURRENCY}${sale.total.toFixed(2)}`;
  if (navigator.canShare) {
    try {
      const blob = await new Promise(res => canvas.toBlob(res, 'image/png'));
      const file = new File([blob], `receipt-${saleId}.png`, { type: 'image/png' });
      if (navigator.canShare({ files: [file] })) {
        await navigator.share({ files: [file], title: `Receipt — ${SHOP_NAME}`, text: message });
        return;
      }
    } catch (e) {
      if (e.name === 'AbortError') return; // user cancelled — don't fall through to wa.me
    }
  }
  // Fallback: wa.me deep link
  const phone = (sale.customer_phone || WHATSAPP_NUMBER || '').replace(/\D/g, '');
  const text  = encodeURIComponent(message);
  const url   = phone ? `https://wa.me/${phone}?text=${text}` : `https://wa.me/?text=${text}`;
  window.open(url, '_blank', 'noopener,noreferrer');
});
```

### _drawReceipt(sale) — Canvas 2D

Create a canvas, draw receipt, return the canvas element.

**Canvas dimensions:** width = 400, calculate height dynamically after drawing.

**Drawing approach (use measureText for dynamic height calculation):**

Step 1: Create an offscreen canvas (400 × 100) to measure text heights.
Step 2: Calculate total height needed: header (~120px) + items (30px × N) + totals (~90px) + footer (~50px) + padding.
Step 3: Create final canvas at (400 × calculatedHeight).
Step 4: Draw all sections.

**Color/font constants (from shop.config.js):**
```js
const BRAND = THEME_COLOR;      // primary blue
const BG    = '#ffffff';
const TEXT  = '#1a1a1a';
const MUTED = '#6b7280';
const LINE  = '#e5e7eb';
```

**Header section (y starts at 24):**
- Background fill: white for entire canvas
- Shop name: `font = 'bold 22px system-ui'`, `fillStyle = TEXT`, centered at x=200
- If `LOGO_URL` is non-empty, draw logo image (drawImage, 48×48, centered above shop name). Handle async load: use `await new Promise(res => { img.onload=res; img.src=LOGO_URL; })` before drawing.
- Date line: formatted with `new Date(sale.timestamp?.toDate?.() ?? Date.now()).toLocaleString(LOCALE)` — `font = '13px system-ui'`, `fillStyle = MUTED`, centered
- Sale ID: `font = 'bold 14px monospace'`, `fillStyle = BRAND`, centered. Format: `Bill #${saleId}`
- Thin horizontal rule (stroke LINE color, lineWidth=1) below header

**Items section:**
- Column header row: "ITEM", "QTY", "PRICE", "TOTAL" — `font = 'bold 12px system-ui'`, `fillStyle = MUTED`
  - Columns at x: 16 (left), 210 (right-align qty), 270 (right-align price), 384 (right-align total)
- For each item in `sale.items`:
  - Item name (truncate to ~24 chars with `…` if longer): x=16, `fillStyle = TEXT`, `font = '14px system-ui'`
  - Qty: right-aligned at x=220, `font = '14px system-ui'`
  - Price: `${CURRENCY}${item.price.toFixed(2)}`, right-aligned at x=300
  - Line total: `${CURRENCY}${item.line_total.toFixed(2)}`, right-aligned at x=390
  - Alternate row background: light gray `#f9fafb` every other row (y to y+28)
- Thin horizontal rule after items

**Totals section:**
- Subtotal row: label "Subtotal" at x=220, value right-aligned at x=390; `font = '13px system-ui'`, `fillStyle = MUTED`
- Discount row (only if `sale.discount > 0`): label "Discount" at x=220, value `−${CURRENCY}${sale.discount.toFixed(2)}` right-aligned at x=390, `fillStyle = '#ef4444'`
- Thick horizontal rule (lineWidth=2, stroke TEXT)
- Total row: label "TOTAL" at x=220, value `${CURRENCY}${sale.total.toFixed(2)}` right-aligned at x=390; `font = 'bold 18px system-ui'`, `fillStyle = TEXT`

**Footer section:**
- Thin rule
- Text: `Thank you for shopping at ${SHOP_NAME}!` — `font = '12px system-ui'`, `fillStyle = MUTED`, centered
- If `sale.customer_phone`: add line `Customer: ${sale.customer_phone}` below, same style

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL → saleId | saleId extracted from hash URL (user-controlled), used in Firestore doc path |
| Firestore sale doc → canvas | Item names from Firestore drawn to canvas via fillText |
| Canvas PNG → Web Share / wa.me | PNG blob shared externally; phone number included in wa.me URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | saleId in URL | accept | Firestore Security Rules enforce per-shop isolation — user can only read their shop's sales. URL manipulation just returns "Receipt not found" for non-existent IDs. |
| T-04-02 | Information Disclosure | saleId in URL | accept | saleId is a non-secret counter ID. No PII in the ID itself. Sale data protected by Firestore rules. |
| T-04-03 | Injection (XSS) | item.name → canvas | accept | Canvas `ctx.fillText()` renders text as pixels — no HTML parsing, no script execution. XSS via canvas is not possible. |
| T-04-04 | Injection | customer_phone → wa.me URL | mitigate | Strip all non-digit characters: `.replace(/\D/g, '')` before building wa.me URL. Prevents URL injection via phone field. |
| T-04-05 | Information Disclosure | PNG file via Web Share API | accept | User explicitly initiates share action. Shared to apps the user chooses. Standard browser share sheet. |
| T-04-06 | Open Redirect | window.open(wa.me) | mitigate | Always use `noopener,noreferrer` window features. URL is constructed from known template, not user-supplied raw string. |
</threat_model>
