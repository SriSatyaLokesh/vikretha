  <how-to-verify>
    Serve the app with a local static server (e.g. `npx serve .` or VS Code Live Server) and open it in the browser.

    **UAT checklist (tick each):**

    1. **SVG area chart renders**
       - Navigate to `#/dashboard`
       - Confirm a chart with a gradient area fill, a line, and dots appears under "Last 7 Days"
       - Confirm it is NOT the old CSS flexbox bars

    2. **Day labels + proportional area**
       - Chart shows 3-letter day labels on the x-axis (e.g. Mon, Tue, Wed…)
       - The fill area is proportional to the revenue values (taller = more revenue)
       - Today's label is in orange / bold

    3. **Hover tooltip (desktop)**
       - Hover over any dot — a browser native tooltip shows the day name and exact revenue (e.g. "Wed: ₹1,234.50")

    4. **Live today counter**
       - Open the dashboard in **two browser tabs** (both pointing to the same shop)
       - In Tab 2, record a new sale (go to Billing, add an item, submit)
       - Switch to Tab 1 — the "Today" stat card should update (revenue + sale count) **without refreshing the page**

    5. **Mobile responsive**
       - Resize the browser window to ≤ 430px (or use DevTools device simulation — e.g. iPhone 14 Pro)
       - Confirm the chart fills the card width with no horizontal scrollbar / overflow

    6. **Zero-revenue days**
       - If any of the last 7 days had zero sales, confirm a dot is visible at the chart baseline (the chart should not appear broken or have a gap)

    Type **"approved"** to complete Phase 19, or describe any issues found.
