Create `lib/svg-chart.js` as a pure ES module with a single exported function `drawAreaChart(container, data, options)`. No external dependencies. No build step.

**Function signature:**
```javascript
/**
 * Draws a responsive SVG area chart into container.
 * @param {HTMLElement} container - cleared and filled with <svg>
 * @param {Array<{label: string, value: number, isToday?: boolean}>} data - 7 items
 * @param {{ currencyFormatter?: (v: number) => string }} options
 */
export function drawAreaChart(container, data, options = {}) { ... }
```

**SVG layout constants (at module top):**
```javascript
const W = 320, H = 120;
const PAD = { top: 14, right: 10, bottom: 24, left: 6 };
const CHART_W = W - PAD.left - PAD.right;
const CHART_H = H - PAD.top - PAD.bottom;
```

**Implementation steps inside drawAreaChart:**
1. Guard: `if (!container || !Array.isArray(data) || data.length === 0) return;`
2. `const fmt = options.currencyFormatter || (v => String(v));`
3. Compute `maxVal = Math.max(...data.map(d => d.value), 1)` — prevents divide-by-zero when all values are 0.
4. Compute point coordinates:
   ```javascript
   const xStep = data.length > 1 ? CHART_W / (data.length - 1) : CHART_W;
   const xOf = i => PAD.left + i * xStep;
   const yOf = v => PAD.top + CHART_H - (v / maxVal) * CHART_H;
   const pts = data.map((d, i) => ({ x: xOf(i), y: yOf(d.value) }));
   ```
5. Build SVG path strings:
   ```javascript
   const linePath = pts.map((p, i) => (i === 0 ? `M${p.x},${p.y}` : `L${p.x},${p.y}`)).join(' ');
   const areaPath = linePath
     + ` L${pts[pts.length - 1].x},${PAD.top + CHART_H}`
     + ` L${pts[0].x},${PAD.top + CHART_H} Z`;
   ```
6. XSS guard for tooltip text (defense in depth):
   ```javascript
   const esc = s => String(s).replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;');
   ```
7. Build grid lines at 25%, 50%, 75% chart height:
   ```javascript
   const gridLines = [0.25, 0.5, 0.75].map(f => {
     const y = PAD.top + CHART_H * (1 - f);
     return `<line x1="${PAD.left}" y1="${y.toFixed(1)}" x2="${W - PAD.right}" y2="${y.toFixed(1)}"/>`;
   }).join('');
   ```
8. Build dots (today dot: r=4.5, stroke=2; normal dot: r=3):
   ```javascript
   const dots = pts.map((p, i) => {
     const d = data[i];
     const r = d.isToday ? 4.5 : 3;
     const cls = d.isToday ? 'svg-chart-dot svg-chart-dot--today' : 'svg-chart-dot';
     const tooltip = `${esc(d.label)}: ${esc(fmt(d.value))}`;
     return `<circle class="${cls}" cx="${p.x.toFixed(1)}" cy="${p.y.toFixed(1)}" r="${r}"><title>${tooltip}</title></circle>`;
   }).join('');
   ```
9. Build x-axis labels:
   ```javascript
   const labels = data.map((d, i) => {
     const x = xOf(i).toFixed(1);
     const cls = d.isToday ? 'svg-chart-axis-label svg-chart-axis-label--today' : 'svg-chart-axis-label';
     return `<text class="${cls}" x="${x}" y="${H - 5}" text-anchor="middle">${esc(d.label)}</text>`;
   }).join('');
   ```
10. Set `container.innerHTML` to the full SVG string:
    ```javascript
    container.innerHTML = `<svg viewBox="0 0 ${W} ${H}" preserveAspectRatio="none"
        aria-label="7-day revenue chart" role="img">
      <defs>
        <linearGradient id="svg-chart-gradient" x1="0" y1="0" x2="0" y2="1">
          <stop offset="0%" stop-color="var(--primary)" stop-opacity="0.28"/>
          <stop offset="100%" stop-color="var(--primary)" stop-opacity="0.02"/>
        </linearGradient>
      </defs>
      <g class="svg-chart-grid">${gridLines}</g>
      <path class="svg-chart-area" d="${areaPath}"/>
      <path class="svg-chart-line" d="${linePath}"/>
      <g class="svg-chart-dots">${dots}</g>
      <g class="svg-chart-axis">${labels}</g>
    </svg>`;
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Firestore → SVG render | `daily_summary.revenue` (number) and `daily_summary.count` (number) flow into SVG path math and tooltip text |
| Static labels → SVG text | Day labels come from a static `DAY_LABELS` array (not from Firestore or user input) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Injection (XSS) | `lib/svg-chart.js` innerHTML | mitigate | `esc()` helper escapes `&`, `<`, `>`, `"` in all values inserted into SVG markup. Revenue is a number (`.toLocaleString()` output is safe); day labels come from a static array. Defense in depth: esc() applied regardless. |
| T-19-02 | Denial of Service | `drawAreaChart` — divide by zero | mitigate | `maxVal = Math.max(...data.map(d => d.value), 1)` ensures denominator is always ≥ 1, even when all 7 days have zero revenue. |
| T-19-03 | Information Disclosure | SVG DOM — revenue values in `<title>` | accept | Revenue data is already on the authenticated dashboard page. Tooltips expose no additional attack surface; data is the owner's own. |
| T-19-04 | Tampering | `snap.data()` fast-path | accept | Snapshot is authenticated via Firebase SDK (same auth session). No additional trust check needed. |
</threat_model>
