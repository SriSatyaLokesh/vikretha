**Module state:**
```js
let _allDocs  = [];     // all loaded QueryDocumentSnapshots
let _filtered = [];     // after client-side search filter
let _lastDoc  = null;   // pagination cursor
let _loading  = false;
let _fromDate = null;   // ISO date string 'YYYY-MM-DD' or null
let _toDate   = null;
```

**`_buildQuery(afterDoc = null)` helper:**
- Base: `query(collection(db,'shops',SHOP_ID,'sales'), orderBy('timestamp','desc'), limit(25))`
- If `_fromDate`: add `where('timestamp','>=', Timestamp.fromDate(new Date(_fromDate+'T00:00:00')))`
- If `_toDate`:   add `where('timestamp','<=', Timestamp.fromDate(new Date(_toDate+'T23:59:59')))`
- If `afterDoc`:  add `startAfter(afterDoc)` (for Load More)

**`_loadSales(reset = true)` async function:**
- If `reset`: `_allDocs = []; _lastDoc = null;`
- Show `#rpt-loading`, hide `#rpt-empty`
- `const snap = await getDocs(_buildQuery(reset ? null : _lastDoc))`
- Append `snap.docs` to `_allDocs`; update `_lastDoc = snap.docs.at(-1) ?? null`
- Apply `_applySearch()` to compute `_filtered`
- Render rows via `_renderRows()`
- Show/hide Load More: visible if `snap.docs.length === 25`

**`_applySearch()` function:**
- `const q = document.getElementById('rpt-search')?.value.trim().toLowerCase() ?? ''`
- If empty: `_filtered = [..._allDocs]`
- Else filter: `data.saleId?.toLowerCase().includes(q) || data.customer_name?.toLowerCase().includes(q) || data.customer_phone?.includes(q)`

**`_renderRows()` function:**
- Build `<tr>` rows from `_filtered`
- Date cell: `new Date(data.timestamp?.toDate?.() ?? Date.now()).toLocaleString(LOCALE, {dateStyle:'short', timeStyle:'short'})`
- Sale ID cell: `data.saleId`
- Customer cell: `data.customer_name || '—'`
- Amount cell: `CURRENCY + data.total.toLocaleString(LOCALE, {minimumFractionDigits:2})`
- If `data.editedAt` exists: append ` <span class="rpt-amended-badge" title="Amended">✏️</span>` inside the amount cell
- Each `<tr>` gets `data-doc-id="${escapeHtml(doc.id)}"` attribute
- If `_filtered.length === 0`: show `#rpt-empty`

**Event bindings (after innerHTML):**
- `#rpt-filter-btn` click → `_fromDate = rptFrom.value||null; _toDate = rptTo.value||null; _loadSales(true)`
- `#rpt-clear-btn` click → clear date inputs + `_fromDate = _toDate = null; _loadSales(true)`
- `#rpt-search` input event (debounced 300ms) → `_applySearch(); _renderRows()`
- `#rpt-load-more` click → `_loadSales(false)`
- Delegated click on `#rpt-tbody` → if `e.target.closest('tr[data-doc-id]')`, call `_openDetail(docId)`

Do NOT set any default date range — load the latest 25 sales on mount.

**Security:** All data inserted into innerHTML must use `escapeHtml()`. The `data-doc-id` attribute is used only internally for lookup, never rendered unsanitized to the user.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Firestore → reports.js | Sale data comes from Firestore; treated as trusted (already authenticated + rules-enforced) but user-generated strings (customer_name, saleId) must be HTML-escaped before innerHTML |
| User search input → DOM | Search field value is compared with `includes()` — no eval, no SQL, no innerHTML insertion of raw input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Injection (XSS) | customer_name, saleId in innerHTML | mitigate | All Firestore string fields passed through `escapeHtml()` before insertion |
| T-15-02 | Injection (XSS) | Search input value | mitigate | Search field value used only with `.includes()` comparison, never injected to DOM |
| T-15-03 | Injection (hash) | View Receipt link with docId | mitigate | `encodeURIComponent(docId)` before hash assignment |
| T-15-04 | Information Disclosure | Sales data visible to all staff | accept | All staff are authenticated + authorized via Firestore rules; business data is appropriate for all staff |
</threat_model>
