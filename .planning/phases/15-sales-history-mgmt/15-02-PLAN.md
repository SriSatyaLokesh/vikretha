  <name>Task 1: Firestore rules — add owner sales update permission</name>
  <files>firestore.rules</files>
  <action>
Open `firestore.rules`. Find the sales rule block:

```
// ── sales ──────────────────────────────────────────────────────────
match /shops/{shopId}/sales/{saleId} {
  allow read: if isAuthorized(shopId);
  allow create: if isAuthorized(shopId)
    && request.resource.data.keys().hasAll(['saleId', 'timestamp', 'items', 'total'])
    && request.resource.data.total >= 0;
  // No updates or deletes — immutable
  allow update, delete: if false;
}
```

Replace the `allow update, delete: if false;` line with TWO separate rules:

```
// Owner can amend a sale (correction with audit trail). Delete still blocked.
allow update: if isAuthorized(shopId)
  && getRole(shopId) == 'owner'
  && request.resource.data.keys().hasAll(['saleId', 'timestamp', 'items', 'total',
      'editedAt', 'editedBy', 'originalTotal', 'amendedTotal'])
  && request.resource.data.total >= 0
  && request.resource.data.amendedTotal >= 0;
allow delete: if false;
```

**Why separate rules (not combined):** Each `allow` is independently evaluated. Separating create/update/delete makes intent explicit and audit-friendly.

**Security rationale:**
- `getRole(shopId) == 'owner'` — only owner role (not admin/cashier) can edit bills
- Required fields validated (`editedAt`, `editedBy`, `originalTotal`, `amendedTotal`) — prevents partial writes that would corrupt audit trail
- `total >= 0` and `amendedTotal >= 0` — prevents negative total writes
- Delete remains blocked for all — immutable audit log

**Step 2 — `_injectEditZone(data, docId)` async function:**

```js
async function _injectEditZone(data, docId) {
  const zone = document.getElementById('rpt-edit-zone');
  if (!zone) return;

  // Role check — only render for owner
  const configRef = doc(db, 'shops', SHOP_ID, 'config', 'main');
  const cfgSnap   = await getDoc(configRef);
  const roles     = cfgSnap.exists() ? (cfgSnap.data().staff_roles || {}) : {};
  const isOwner   = roles[auth.currentUser?.email] === 'owner';
  if (!isOwner) return; // cashier/admin sees nothing

  zone.innerHTML = `
    <button id="rpt-edit-btn" class="btn btn-warning btn-sm" style="margin-top:4px;">
      ✏️ Edit Bill
    </button>
    <div id="rpt-edit-form" style="display:none;margin-top:12px;">
      <p class="rpt-edit-warning">
        ⚠️ Editing a bill is permanent and logged. Adjust only to correct errors.
      </p>
      <table class="rpt-edit-items-table" id="rpt-edit-items">
        <thead><tr>
          <th>Item name</th><th>Qty</th><th>Unit price (${CURRENCY})</th><th></th>
        </tr></thead>
        <tbody id="rpt-edit-tbody"></tbody>
      </table>
      <button type="button" id="rpt-add-row" class="btn btn-ghost btn-sm" style="margin-top:4px;">
        + Add row
      </button>
      <div class="rpt-edit-discount" style="margin-top:10px;">
        <label for="rpt-edit-disc">Discount (${CURRENCY})</label>
        <input type="number" id="rpt-edit-disc" min="0" step="0.01"
               class="rpt-date-input" style="width:100px;"
               value="${data.discount ?? 0}">
      </div>
      <div class="rpt-edit-totals" id="rpt-edit-totals" style="margin-top:8px;font-size:0.9rem;"></div>
      <div class="rpt-edit-actions" style="display:flex;gap:8px;margin-top:14px;flex-wrap:wrap;">
        <button type="button" id="rpt-save-edit" class="btn btn-primary">Save changes</button>
        <button type="button" id="rpt-cancel-edit" class="btn btn-ghost">Cancel</button>
      </div>
      <div id="rpt-edit-error" class="rpt-edit-error" style="display:none;"></div>
    </div>`;

  // Populate item rows from current data
  _populateEditRows(data.items || []);
  _recalcEditTotals();

  // Event: toggle form
  zone.querySelector('#rpt-edit-btn').addEventListener('click', () => {
    const form = zone.querySelector('#rpt-edit-form');
    form.style.display = form.style.display === 'none' ? 'block' : 'none';
  });

  // Event: add row
  zone.querySelector('#rpt-add-row').addEventListener('click', () => {
    _addEditRow({ name: '', price: 0, qty: 1, unit: '' });
    _recalcEditTotals();
  });

  // Event: live recalc on any input change in table or discount
  zone.addEventListener('input', e => {
    if (e.target.matches('#rpt-edit-tbody input, #rpt-edit-disc')) _recalcEditTotals();
  });

  // Event: remove row (delegated)
  zone.querySelector('#rpt-edit-tbody').addEventListener('click', e => {
    if (e.target.closest('.rpt-rm-row')) {
      e.target.closest('tr').remove();
      _recalcEditTotals();
    }
  });

  // Event: cancel
  zone.querySelector('#rpt-cancel-edit').addEventListener('click', () => {
    zone.querySelector('#rpt-edit-form').style.display = 'none';
  });

  // Event: save
  zone.querySelector('#rpt-save-edit').addEventListener('click', () =>
    _saveEdit(data, docId, zone));
}
```

**`_populateEditRows(items)` function:**
- For each item, call `_addEditRow(item)` to create a table row.

**`_addEditRow(item)` function:**
- Append a `<tr>` to `#rpt-edit-tbody`:
```html
<tr>
  <td><input type="text"   class="rpt-edit-name"  value="${escapeHtml(item.name)}"  style="width:100%;box-sizing:border-box;"></td>
  <td><input type="number" class="rpt-edit-qty"   value="${item.qty}"   min="1" step="1"    style="width:60px;"></td>
  <td><input type="number" class="rpt-edit-price" value="${item.price}" min="0" step="0.01" style="width:90px;"></td>
  <td><button class="rpt-rm-row btn-icon" title="Remove" aria-label="Remove row">×</button></td>
</tr>
```
  (Preserve hidden fields as JS variables, not DOM inputs: `item.unit`, `item.size_label`, `item.size_key`, `item.item_id`, `item.adhoc`. Store as `data-*` attributes on `<tr>` for retrieval on save.)

**`_recalcEditTotals()` function:**
- Read all rows in `#rpt-edit-tbody`, compute `subtotal = sum(qty * price)`
- Read discount from `#rpt-edit-disc`, clamp to [0, subtotal]
- `newTotal = subtotal - discount`
- Update `#rpt-edit-totals`:
  ```
  Subtotal: ₹X.XX  |  Discount: ₹X.XX  |  Total: ₹X.XX
  ```

**`_saveEdit(originalData, docId, zone)` async function:**
- Disable `#rpt-save-edit`, show "Saving…"
- Read rows: parse each `<tr>` in `#rpt-edit-tbody` to build `newItems[]`
  - name = `.rpt-edit-name` input value (trim, required — skip empty-name rows)
  - qty  = parseInt(`.rpt-edit-qty`) — must be ≥ 1
  - price = parseFloat(`.rpt-edit-price`) — must be ≥ 0
  - Restore `unit`, `size_label`, `size_key`, `item_id`, `adhoc` from `data-*` attributes on `<tr>`
- Validate: at least 1 item; all qty ≥ 1; all price ≥ 0. If invalid, show error in `#rpt-edit-error` and re-enable button.
- Compute `newSubtotal`, `newDiscount` (clamped), `newTotal`
- Determine `originalTotal`: use `originalData.originalTotal ?? originalData.total` (preserve first original if already amended)
- Call:
  ```js
  await updateDoc(
    doc(db, 'shops', SHOP_ID, 'sales', docId),
    {
      items:         newItems,
      subtotal:      newSubtotal,
      discount:      newDiscount,
      total:         newTotal,
      editedAt:      serverTimestamp(),
      editedBy:      auth.currentUser.email,
      originalTotal: originalTotal,
      amendedTotal:  newTotal
    }
  );
  ```
- On success: update the in-memory doc in `_allDocs` with merged data, re-render detail panel via `_renderDetailPanel(updatedData, docId)` and re-call `_injectEditZone(updatedData, docId)`.
- On error: show error message in `#rpt-edit-error`, re-enable button. Do NOT swallow errors silently.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Edit form inputs → Firestore updateDoc | User-controlled item names, quantities, prices flow into Firestore |
| Client role check → UI visibility | Role check controls "Edit Bill" button visibility; Firestore rules enforce server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-05 | Tampering | Cashier editing bills by bypassing client-side role check | mitigate | Firestore rules require `getRole(shopId) == 'owner'` — server enforced, cannot be bypassed client-side |
| T-15-06 | Tampering | Negative total via edit (e.g., setting price to -999) | mitigate | Firestore rule validates `total >= 0` and `amendedTotal >= 0`; client-side also clamps discount |
| T-15-07 | Tampering | Omitting audit fields on updateDoc | mitigate | Firestore rule requires `editedAt`, `editedBy`, `originalTotal`, `amendedTotal` via `hasAll()` check |
| T-15-08 | Injection (XSS) | Item name inputs in edit form → innerHTML on re-render | mitigate | All item names pass through `escapeHtml()` before any innerHTML use |
| T-15-09 | Elevation of Privilege | Admin trying to edit bills | mitigate | Rule checks `getRole == 'owner'` not `isOwnerOrAdmin` — admins are explicitly excluded from bill editing |
| T-15-10 | Repudiation | No audit trail for edits | mitigate | `editedAt`, `editedBy`, `originalTotal` stored on sale doc; immutable by subsequent cashier writes |
</threat_model>
