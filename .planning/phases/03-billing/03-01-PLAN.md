**`_renderProductList(query)` function:**
- Filters `_inventory` case-insensitively: `item.name.toLowerCase().includes(query.toLowerCase())`
- If `_inventory.length === 0`: show `<div class="empty-state"><p>No products in inventory.</p><p style="font-size:0.75rem;margin-top:4px;">Add items via Inventory tab (coming soon).</p></div>`
- If query matches nothing: show `<div class="empty-state"><p>No results for "${escapeHtml(query)}"</p></div>`
- Else: render list of product cards — each card shows name, unit, price, stock badge, "+ Add" button
- If item already in cart: show qty spinner (− N +) and "In Cart" indicator instead of "+ Add" button
- Render into `#product-list`

**`_renderCartItems()` function:**
- Renders each cart item in `_cart` as a row: name, qty (−/+), line total (price × qty), remove (×) button
- Show/hide `#cart-section` based on `_cart.size > 0`
- Calls `_updateTotals()`

**`_updateTotals()` function:**
- Subtotal = sum of all `price × qty` in `_cart`
- Discount mode stored in module-level `_discMode` ('pct' or 'inr', default 'pct')
- Discount value from `#discount-input` (parseFloat, default 0)
- Total = pct mode: `subtotal * (1 - disc/100)`; inr mode: `Math.max(0, subtotal - disc)`
- Update `#subtotal-display`, `#total-display`, `#discount-display`, show/hide `#discount-line`

**Event listeners (attach after innerHTML):**

**XSS safety:** Use `escapeHtml()` (implement locally — same as auth.js: escapes `&`, `<`, `>`, `"`) for all product names and phone numbers rendered via innerHTML. Prices and quantities are numbers — no escaping needed.

**`_loadInventory(container)` function:**
```javascript
function _loadInventory(container) {
  const inventoryRef = collection(db, 'shops', SHOP_ID, 'inventory');
  _unsubInventory = onSnapshot(inventoryRef, (snapshot) => {
    _inventory = snapshot.docs.map(d => ({ id: d.id, ...d.data() }));
    const searchInput = container.querySelector('#product-search');
    _renderProductList(searchInput?.value || '');
  }, (err) => {
    console.error('[Billing] inventory snapshot error', err);
  });
}
```

**`_generateSaleId()` async function:**
- Use `runTransaction` to read `/shops/{SHOP_ID}/counters/sales_counter`
- If doc exists: `nextSeq = data.last_seq + 1`; else `nextSeq = 1`
- If doc exists: `transaction.update(counterRef, { last_seq: nextSeq })`; else `transaction.set(counterRef, { last_seq: nextSeq })`
- Format: `${today}-${String(nextSeq).padStart(4, '0')}` where `today = new Date().toISOString().slice(0,10).replace(/-/g,'')`
- Returns the formatted saleId string

**`_generateOfflineSaleId()` function (sync, no Firestore):**
```javascript
function _generateOfflineSaleId() {
  const now = new Date();
  const date = now.toISOString().slice(0, 10).replace(/-/g, '');
  const ms = now.getTime().toString(36);
  return `${date}-OFF-${ms.toUpperCase()}`;
}
```

**`_handleSubmit(container)` async function:**
1. Validate cart not empty. If empty: show error in `#submit-error`.
2. Get discount value, mode, phone from DOM.
3. Calculate subtotal and total (same logic as `_updateTotals`).
4. Disable `#submit-sale-btn` immediately (set `disabled=true`, text = spinner HTML).
5. Try `saleId = await _generateSaleId()`. Catch network error → `saleId = _generateOfflineSaleId()`.
6. Build `cartArray` from `_cart`: `[...cart.values()].map(item => ({ item_id: item.id, name: item.name, price: item.price, qty: item.qty, line_total: item.price * item.qty }))`.
7. Create `writeBatch(db)`.
8. `batch.set(saleRef, { saleId, timestamp: serverTimestamp(), items: cartArray, subtotal, discount: discountAmount, total, customer_phone: phone || null, created_by: auth.currentUser?.email || null })` — saleRef = `doc(db, 'shops', SHOP_ID, 'sales', saleId)`.
9. For each cart item: `batch.update(doc(db, 'shops', SHOP_ID, 'inventory', item.item_id), { stock: increment(-item.qty) })`.
10. `batch.set(summaryRef, { today_count: increment(1), today_revenue: increment(total), week_count: increment(1), week_revenue: increment(total), month_count: increment(1), month_revenue: increment(total), last_updated: serverTimestamp() }, { merge: true })` — summaryRef = `doc(db, 'shops', SHOP_ID, 'summary')`.
11. Call `await batch.commit()`.
12. On success: call `_showConfirmation(container, { saleId, total, cartArray })`.
13. On error: re-enable submit button, show error in `#submit-error`: `'Failed to save sale. Please try again.'`.

**`_showConfirmation(container, { saleId, total, cartArray })` function:**
- Replace `container.innerHTML` with a confirmation screen:
  - "✅ Sale Recorded" heading
  - Sale ID in large monospace font: `#${saleId}`
  - Total amount: `${CURRENCY}${total.toLocaleString(LOCALE, {minimumFractionDigits:2})}`
  - Item count: `${cartArray.length} item(s)`
  - Sync badge: `<span class="sync-badge pending">⏳ Syncing...</span>` (id="sync-badge")
  - "New Sale" button → calls `render(container)` to reset the screen
  - "View Receipt" button → `window.location.hash = '#/receipt/' + saleId` (Phase 4 builds this)
- After rendering: subscribe to `onSnapshot(saleRef, { includeMetadataChanges: true }, (snap) => { if (!snap.metadata.hasPendingWrites) { badge.className='sync-badge synced'; badge.textContent='✓ Synced'; unsubscribe(); } })`.
- Store unsubscribe in local `const unsubSync = onSnapshot(...)` and call `unsubSync()` inside the callback after updating badge.

**Security:** `auth.currentUser?.email` is always a Firebase-validated email — safe to write to Firestore. Customer phone is stored as-is (no validation beyond field type). Never render `customer_phone` back into innerHTML without `escapeHtml()`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser JS → Firestore | All writes pass through Firestore SDK; Security Rules enforce authorization |
| User input → DOM (innerHTML) | Product names from Firestore, customer phone from user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | E (XSS) | Product names from Firestore rendered in innerHTML | mitigate | Use `escapeHtml()` on all string values rendered via innerHTML (product names, phone numbers); prices/quantities are numbers, no escaping needed |
| T-03-02 | D (DoS / Duplicate) | Double-tap on Submit Sale creates duplicate sale | mitigate | Disable submit button immediately on click; re-enable only on error; `runTransaction` uses server-side atomicity |
| T-03-03 | T (Tampering) | Negative qty or discount larger than subtotal | mitigate | HTML `min="0"` on qty and discount inputs; `Math.max(0, subtotal - disc)` in total calculation prevents negative totals |
| T-03-04 | S (Spoofing) | Unauthorized user submits sale to another shop | accept | Firestore Security Rules `isAuthorized(shopId)` from Phase 2 rejects unauthorized writes at the server; client-side SHOP_ID is fixed in shop.config.js |
| T-03-05 | I (Disclosure) | Inventory data from other shops | accept | Firestore Security Rules already enforce per-shop isolation (Phase 2); no cross-shop queries possible |
| T-03-06 | T (Tampering) | Sale doc missing required fields if code modified | accept | Single-tenant self-hosted app; shop owner is the developer; no multi-user trust boundary at JS level |
</threat_model>
