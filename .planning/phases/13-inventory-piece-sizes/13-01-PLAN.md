<objective>
Rewrite the inventory add/edit modals to enforce unit type (Pieces vs Other) and support
size variant configuration for piece-type items.

Purpose: The current unit field is a free-text input. Users are accidentally entering CG/litres
for piece items. This change makes "Pieces" the default locked unit and adds size variants
(label, width, PSI, per-size stock) for piece items.

Output: Updated modules/inventory.js with custom modals, new Firestore schema fields, and
updated low-stock logic.
</objective>

<task type="auto">
  <name>Task 1: Rewrite _showAddModal with custom piece/sizes UI</name>
  <files>modules/inventory.js</files>
  <action>
Replace the entire `_showAddModal` function with a new implementation that builds a
custom bottom-sheet overlay (same visual style as `_showModal` — same overlay, sheet, header,
close button, save button, error paragraph patterns).

**Unit type selector (segmented toggle, same style as `#sort-name`/`#sort-low`):**
- Two buttons: `[Pieces]` (active by default) and `[Other]`
- When "Pieces" selected: `unit` is 'pcs'; hide the custom unit text input
- When "Other" selected: show a text input for custom unit (placeholder: 'e.g. kg / ltr / CG')
  `unit` value = text input value

**Has sizes? (checkbox row, visible only when Pieces is selected):**
- Checkbox labeled "Has sizes?" (`id="inv-has-sizes"`)
- When checked: show the sizes section; when unchecked: hide sizes section
- When "Other" selected: force-uncheck and hide this checkbox

**Quantity field behavior:**
- When `hasSizes` is false: show standard "Total stock" number input (`id="inv-qty"`)
- When `hasSizes` is true: hide the qty input (stock lives per-size); show the sizes section instead

**Sizes section (visible when Pieces + hasSizes):**
- A `<div id="inv-sizes-list">` that holds size variant rows
- "Add size +" button that appends a new row to the list
- Each row contains:
  - Text input: Size label (e.g. "10x12") — required — `class="size-label-input"`
  - Text input: Width (optional) — `class="size-width-input"` — placeholder "Width (optional)"
  - Text input: PSI (optional) — `class="size-psi-input"` — placeholder "PSI (optional)"
  - Number input: Stock qty — required, min 0 — `class="size-stock-input"` — placeholder "Qty"
  - Remove button ("×") that removes this row
- Row layout: use `display:grid; grid-template-columns: 2fr 1fr 1fr 1fr auto; gap:6px`

**Save handler logic:**
1. Read `unitType` from the active toggle button
2. `unit = unitType === 'pieces' ? 'pcs' : customUnitInput.value.trim()`
3. Validate: name required; unit required (for Other); price > 0; threshold >= 0
4. If `hasSizes`:
   - Read all size rows; validate at least 1 size row exists
   - Each row: label required, stock >= 0
   - Sanitize `sizeKey = label.trim().toLowerCase().replace(/[^a-z0-9]/g, '-').substring(0, 40)`
   - Validate no duplicate sizeKeys
   - Build `sizes` map: `{ [sizeKey]: { label, width: width||null, psi: psi||null, stock: parseInt(stock) } }`
   - Write doc: `{ name, unitType:'pieces', unit:'pcs', price, stock:0, threshold, hasSizes:true, sizes }`
5. If NOT hasSizes (Pieces without sizes) OR Other:
   - `stock = parseInt(inv-qty value)`; validate >= 0
   - Write doc: `{ name, unitType, unit, price, stock, threshold, hasSizes:false }`

Use `addDoc(collection(db, 'shops', SHOP_ID, 'inventory'), docData)` as before.

**XSS safety:** All user input written to DOM must use `.textContent =` or `escapeHtml()`.
Never use `.innerHTML = userInput` directly.

**Styling:** Use CSS custom properties already in the design system:
`--bg-primary`, `--bg-surface`, `--border`, `--border-radius`, `--text-primary`,
`--text-secondary`, `--theme-color`, `--danger`.
Size rows use `--bg-surface` background, `8px` border-radius, `8px` padding.
