### 5. Setup (numbered steps — the core section)
Must be thorough enough for < 30 min fork-to-working:

**Step 1 — Fork this repository**
- Fork button on GitHub, then clone locally (optional — GitHub Pages doesn't require it)

**Step 2 — Create a Firebase project**
- Firebase Console → Add project → choose a name → disable Google Analytics (optional) → Create project
- Enable Firestore: Firestore Database → Create database → Start in **test mode** (rules added in Step 5) → choose region closest to users (e.g., asia-south1 for India)
- Enable Authentication: Authentication → Sign-in method → Email/Password → Enable → Save

**Step 3 — Get your Firebase config**
- Project Settings (gear icon) → General → Your apps → Web app → Register app (if not done) → copy the config object

**Step 4 — Configure shop.config.js**
- Open `shop.config.js` in your fork
- Replace the `FIREBASE_CONFIG` block with your config values
- Set `SHOP_NAME`, `SHOP_ID`, `CURRENCY`, `LOCALE`, `WHATSAPP_NUMBER` (optional), `THEME_COLOR`
- Commit and push to your fork

**Step 5 — Set Firestore Security Rules**
- Copy the full contents of `firestore.rules` from this repository
- Firebase Console → Firestore Database → Rules tab → paste → Publish

**Step 6 — Enable GitHub Pages**
- Repository Settings → Pages → Source: Deploy from a branch → Branch: main → Folder: / (root) → Save
- Site publishes at `https://{your-username}.github.io/{repo-name}/`

**Step 7 — First sign-in**
- Visit your GitHub Pages URL
- Click "Create Account" → enter your email and a password → the app bootstraps your shop config automatically
- You're ready to use Vikretha

### 6. Configuration Reference
Table documenting every field in shop.config.js:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| FIREBASE_CONFIG.apiKey | string | — | Firebase API key (from Project Settings) |
| FIREBASE_CONFIG.authDomain | string | — | `{projectId}.firebaseapp.com` |
| FIREBASE_CONFIG.projectId | string | — | Firebase project ID |
| FIREBASE_CONFIG.storageBucket | string | — | `{projectId}.firebasestorage.app` |
| FIREBASE_CONFIG.messagingSenderId | string | — | Messaging sender ID |
| FIREBASE_CONFIG.appId | string | — | Firebase app ID |
| SHOP_NAME | string | "My Shop" | Displayed in the header and on receipts |
| SHOP_ID | string | "shop_001" | Firestore path segment: `/shops/{SHOP_ID}/`. Use URL-safe string (no spaces). |
| CURRENCY | string | "₹" | Currency symbol shown on receipts and reports |
| LOCALE | string | "en-IN" | BCP 47 locale for date/number formatting |
| WHATSAPP_NUMBER | string | "" | Shop WhatsApp number with country code (e.g., `+919876543210`) |
| THEME_COLOR | string | "#2563eb" | Primary colour (hex) — header, buttons, charts |
| LOGO_URL | string | "" | URL to shop logo shown on receipts (leave empty to hide) |
