### B. Sales (past 30 days — 3 to 6 sales per day)
Use the document ID format: `{YYYYMMDD}-{NNN}` (e.g., `20260501-001`).

For each day from 30 days ago to today:

- Each sale picks 1-3 random items from the inventory list above
- sale document fields (all at Firestore REST level):
  - `saleId` (stringValue): `{YYYYMMDD}-{NNN}` (zero-padded NNN)

  - `total` (integerValue): same as subtotal (no discount)

  <action>
Perform a systematic accessibility audit across all modules and the CSS. Fix every issue found.

**Audit checklist (check each module's HTML generation):**

### 1. ARIA labels on icon-only buttons
Search every module for buttons/anchors that contain only an SVG or emoji (no visible text label). Add `aria-label="Descriptive action"` to each.

Common patterns to find:
- Download button in receipt.js → `aria-label="Download receipt"`
- WhatsApp share button in receipt.js → `aria-label="Share via WhatsApp"`
- Remove staff button in settings.js → `aria-label="Remove ${email}"`
- Close modal buttons → `aria-label="Close"`
- Edit/delete buttons in inventory.js → `aria-label="Edit ${item.name}"`, `aria-label="Delete ${item.name}"`
- Export dropdown toggle button in export.js → `aria-label="Export data"`
- Any ☰ hamburger or × close buttons → appropriate labels

### 2. Touch targets (styles/main.css)
Add or verify these rules exist in main.css. Add them at the end in a new section `/* === Section 20: Accessibility === */`:

```css
/* Minimum touch targets per WCAG 2.1 AA (44×44px) */
button,
a,
[role="button"],
input[type="checkbox"],
input[type="radio"],
select {
  min-height: 44px;
  /* Do NOT override existing larger heights */
}

/* Exception: inline text links don't need 44px height */
p a, li a, td a {
  min-height: unset;
}

/* Ensure icon buttons are at least 44×44 */
.btn-icon {
  min-width: 44px;
  min-height: 44px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
}
```

### 3. Focus visibility
Ensure main.css has a visible focus ring. Add if not present:
```css
:focus-visible {
  outline: 2px solid var(--color-primary, #2563eb);
  outline-offset: 2px;
}
```

### 4. Form labels
Check modules that generate form inputs. Verify every `<input>` and `<select>` has either:
- An associated `<label for="...">` element, OR
- `aria-label="..."` attribute, OR
- `aria-labelledby="..."` pointing to a heading

Fix any inputs that have neither.

### 5. Role on non-button interactive elements
If any div/span elements are used as clickable buttons (have onclick handlers), add `role="button"` and `tabindex="0"` if missing.

**NOTE:** Make surgical fixes only — do not restructure modules, do not change visual appearance, do not break existing functionality. Change only the HTML attribute strings and add CSS rules.

**After making changes, verify there are no syntax errors** by checking that no module's template literals have unmatched backticks.
