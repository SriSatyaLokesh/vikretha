<context>
@.planning/PROJECT.md

Key project constraints from TRD:
- No build step — all files served directly from GitHub Pages
- App JS payload < 30 KB (excluding Firebase SDK)
- Mobile-first, primary device: Android smartphone (375–430px viewport)
- Routing: hash-based (#/login, #/dashboard, etc.) — works on GitHub Pages without 404 hacks
- Firebase SDK loaded from: https://www.gstatic.com/firebasejs/10.12.0/ (ES module CDN)
- SheetJS (export) loaded lazily from: https://cdn.sheetjs.com/xlsx-0.20.2/package/xlsx.mjs

CSP allowed origins:
  script-src: 'self' https://www.gstatic.com https://cdn.sheetjs.com
  connect-src: 'self' https://*.googleapis.com https://*.firebaseio.com wss://*.firebaseio.com
  style-src: 'self' 'unsafe-inline'   (needed for CSS custom properties in some browsers)
  img-src: 'self' data: blob:
  font-src: 'self'
</context>

  <action>
Create index.html as the single HTML entry point:
- DOCTYPE html, lang="en"
- <head>:
  - <meta charset="UTF-8">
  - <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">
  - <meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' https://www.gstatic.com https://cdn.sheetjs.com; connect-src 'self' https://*.googleapis.com https://*.firebaseio.com wss://*.firebaseio.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self';">
  - <meta name="theme-color" content="#2563eb"> (overridden by shop.config.js at runtime)
  - <link rel="manifest" href="./manifest.json">
  - <link rel="apple-touch-icon" href="./icons/icon-192.png">
  - <link rel="stylesheet" href="./styles/main.css">
  - <title>Vikretha</title>
- <body>:
  - <div id="app"><!-- Router mounts here --></div>
  - <noscript><p>Enable JavaScript to use Vikretha.</p></noscript>
  - <script type="module" src="./app.js"></script>

Create manifest.json:
{
  "name": "Vikretha",
  "short_name": "Vikretha",
  "description": "Shop management PWA for small businesses",
  "start_url": "./index.html",
  "display": "standalone",
  "orientation": "portrait",
  "background_color": "#ffffff",

  "icons": [

Create .nojekyll: empty file (single blank line)

needed, but real files are preferred). Create: icons/icon-192.png and icons/icon-512.png

feasible, create the icons/ directory with a README placeholder noting icons need replacing.

<task type="auto">
  <name>Task 2: Create shop.config.js</name>
  <files>shop.config.js</files>
  <action>
Create shop.config.js as the single customization file (export named constants, not default):

// ================================================================
// VIKRETHA — SHOP CONFIGURATION
// Edit this file to configure your shop. All settings in one place.
// See README.md for setup instructions.
// ================================================================

// --- Firebase Configuration ---
// Get these values from: Firebase Console → Project Settings → Your apps → Config
export const FIREBASE_CONFIG = {
  apiKey: "YOUR_API_KEY",                          // From Firebase Console
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",  // Replace YOUR_PROJECT_ID
  projectId: "YOUR_PROJECT_ID",                   // Your Firestore project ID
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

// --- Shop Identity ---
export const SHOP_NAME = "My Shop";          // Displayed in header and receipts
export const SHOP_ID = "shop_001";           // Unique ID for Firestore path: /shops/{SHOP_ID}/
                                             // Use a URL-safe string (no spaces)

// --- Localization ---
export const CURRENCY = "₹";                // Currency symbol shown on receipts and reports
export const LOCALE = "en-IN";              // Locale for date/number formatting (e.g., "en-IN", "en-US")

// --- WhatsApp ---
export const WHATSAPP_NUMBER = "";          // Your shop's WhatsApp number with country code
                                             // e.g., "+919876543210" — used for receipt sharing

// --- Branding ---
export const THEME_COLOR = "#2563eb";        // Primary color (hex) — used for header, buttons, charts
export const LOGO_URL = "";                  // URL to shop logo image — shown on receipts (leave empty to hide)

Export as named exports only — no default export. The app imports individual names:
  import { FIREBASE_CONFIG, SHOP_ID, SHOP_NAME, CURRENCY, THEME_COLOR } from './shop.config.js';

Add a brief JSDoc comment above each export group (Firebase Config, Shop Identity, etc.).
