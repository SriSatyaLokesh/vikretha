<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete Phase 1 scaffold:
    - index.html (entry point with CSP, PWA links)
    - shop.config.js (Firebase config template)
    - manifest.json (PWA manifest)
    - styles/main.css (mobile-first CSS shell)
    - .nojekyll (GitHub Pages)
    - lib/firebase-init.js (Firebase SDK init + Firestore persistence)
    - app.js (hash router + auth guard + SW registration)
    - sw.js (service worker — app shell caching)
  </what-built>
  <how-to-verify>
    **Step 1 — Start local server (required for SW + ES modules):**
    ```
    npx serve . -l 8080
    ```
    Or use VS Code Live Server extension → right-click index.html → "Open with Live Server"

    **Step 2 — Open http://localhost:8080 in Chrome**

    **Step 3 — Check Console (F12 → Console):**
    ✅ "[Vikretha] Service worker registered" message visible
    ✅ No red errors (yellow warnings about Firebase config placeholder values are OK)

    **Step 4 — Check Application (F12 → Application):**
    ✅ Service Workers: sw.js listed, status "activated and is running"
    ✅ Manifest: "Vikretha" detected with icons listed
    ✅ Cache Storage: vikretha-v1 cache shows app shell files

    **Step 5 — Test auth guard:**
    ✅ Navigate to http://localhost:8080/#/dashboard
    ✅ App shows login placeholder (NOT dashboard content) — auth guard working

    **Step 6 — Test offline (Network tab → Offline checkbox):**
    ✅ Hard refresh (Ctrl+Shift+R) — app shell still loads from cache

    **Step 7 — Mobile viewport (DevTools → Toggle Device Toolbar → 375px):**
    ✅ App header visible at top
    ✅ Bottom navigation bar visible with 5 icons
    ✅ No horizontal scroll
  </how-to-verify>
  <resume-signal>
    Type "approved" if all 7 steps pass.
    Or describe any issues found (e.g., "console error: X", "nav bar not showing").
  </resume-signal>
</task>
