<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete Phase 2 authentication:
    - modules/auth.js (Phone OTP flow — phone input + OTP confirmation + bootstrapShopConfig)
    - firestore.rules (deny-all default + isAuthorized + bootstrap create rule)
    - modules/settings.js (sign-out button + current user display)
  </what-built>
  <how-to-verify>
    **Prerequisites:**
    Before testing, complete Firebase setup in shop.config.js:
    1. Create/use a Firebase project with Phone Auth enabled
    2. Add your test phone number to Firebase Auth → "Phone numbers for testing" (optional, saves SMS)
    3. Fill in real values in shop.config.js (FIREBASE_CONFIG, SHOP_ID)
    4. Deploy firestore.rules to Firebase Console → Firestore → Rules
    5. Start local server: `npx serve . -l 8080`

    **Step 1 — Unauthenticated guard:**
    ✅ Open http://localhost:8080 → full-screen sign-in card visible
    ✅ Open http://localhost:8080/#/dashboard → stays on login screen (no flash of content)

    **Step 2 — Phone input:**
    ✅ Enter 10-digit number (e.g. 9876543210) and tap "Send OTP"
    ✅ Button shows loading spinner while reCAPTCHA runs
    ✅ UI transitions to OTP entry screen ("Sent to +91 98xxxxxx10")
    ✅ Entering fewer than 10 digits shows validation error (no SMS sent)

    **Step 3 — OTP confirmation:**
    ✅ Enter correct 6-digit OTP → auto-submits on 6th digit
    ✅ App navigates to #/dashboard (shop name in header)
    ✅ Bottom nav visible with 5 tabs

    **Step 4 — Error handling:**
    ✅ Enter wrong OTP → inline error "Wrong OTP. Check the SMS and try again."
    ✅ App stays on OTP screen (no navigation)
    ✅ "Back" button returns to phone input step

    **Step 5 — Session persistence:**
    ✅ Close and reopen browser (or hard refresh Ctrl+Shift+R)
    ✅ App loads directly to #/dashboard without showing login screen

    **Step 6 — Sign out:**
    ✅ Tap Settings tab → sign-out screen shows your phone number
    ✅ Tap "Sign Out" → returns to full-screen login screen

    **Step 7 — Firestore Security Rules (if rules deployed):**
    ✅ After sign-in, open DevTools → Network → check for Firestore requests
    ✅ No 403/PERMISSION_DENIED errors in console
    ✅ /shops/{shopId}/config document visible in Firebase Console

    **Optional — Unauthorized phone test:**
    Remove your phone from authorized_phones in Firebase Console,
    then try to read data → console shows PERMISSION_DENIED (expected).
    Re-add your phone after testing.
  </how-to-verify>
  <resume-signal>
    Type "approved" if all steps pass.
    Or describe any issues found (e.g., "reCAPTCHA error on localhost", "OTP not received").
  </resume-signal>
</task>
