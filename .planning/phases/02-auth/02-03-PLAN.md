<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete Phase 2 authentication:
    - modules/auth.js (Email/Password sign-in, Create Account, Forgot Password, bootstrapShopConfig)
    - firestore.rules (deny-all default + isAuthorized via authorized_emails + bootstrap create rule)
    - modules/settings.js (sign-out button + current user email display)
  </what-built>
  <how-to-verify>
    **Prerequisites:**
    1. Fill real Firebase config in shop.config.js (enable Email/Password in Firebase Console → Authentication → Sign-in method)
    2. Deploy firestore.rules: Firebase Console → Firestore → Rules → paste → Publish
    3. Start local server: `npx serve . -l 8080`

    **Step 1 — Unauthenticated guard:**
    ✅ Open http://localhost:8080 → full-screen sign-in card visible (email + password fields)
    ✅ Open http://localhost:8080/#/dashboard → stays on sign-in (no flash of content)

    **Step 2 — Sign In:**
    ✅ Enter correct email + password → navigates to #/dashboard
    ✅ Enter wrong password → inline error "Wrong password..." (no navigation)
    ✅ Enter unknown email → inline error "No account found..." (no navigation)

    **Step 3 — Create Account:**
    ✅ Tap "Create an account" → create-account form appears
    ✅ Fill email + password + confirm → tap "Create Account" → navigates to dashboard
    ✅ Mismatched passwords → "Passwords do not match" error
    ✅ Firebase Console → Authentication → Users shows new account

    **Step 4 — Forgot Password:**
    ✅ On sign-in screen, tap "Forgot password?" → reset form appears
    ✅ Enter registered email → tap "Send Reset Email" → success message shown
    ✅ Reset email arrives in inbox (check spam folder)
    ✅ Back button returns to sign-in with email pre-filled

    **Step 5 — Session persistence:**
    ✅ Close and reopen browser (hard refresh Ctrl+Shift+R)
    ✅ App loads directly to #/dashboard (no re-login)

    **Step 6 — Sign out:**
    ✅ Tap Settings tab → your email address displayed
    ✅ Tap "Sign Out" → returns to full-screen sign-in screen

    **Step 7 — Firestore Security Rules (if rules deployed):**
    ✅ After sign-in, DevTools → Network → no 403/PERMISSION_DENIED errors
    ✅ /shops/{shopId}/config document visible in Firebase Console with authorized_emails array

    **Optional — Unauthorized email test:**
    Remove your email from authorized_emails in Firebase Console, then sign in →
    console shows PERMISSION_DENIED on Firestore reads (expected).
    Re-add your email after testing.
  </how-to-verify>
  <resume-signal>
    Type "approved" if all steps pass.
    Or describe any issues found (e.g., "sign-in works but create account errors").
  </resume-signal>
</task>
